Overview

In this evaluation framework, you will build the infrastructure to compare FIFA's baseline Elo formula against your three targeted ablation modifications — one at a time. This is a large measurement project that requires you and your team to work through every stage of evaluation from a single RPS number to a full multi-metric comparison table.

The goal is to teach you how to measure fairly. There is no "correct answer" when comparing probabilistic forecasting models — you will be assessed on the rigor of your comparison methodology, the honesty of your reported CIs, and the clarity of your interpretation. Report everything, including results that complicate your narrative.

Don't miss the significance of the ablation structure — the academic claim is that each modification improves the baseline independently. The ablation loop in Lesson 2 is the proof. If a modification hurts one metric while improving another, that is a finding worth discussing, not something to paper over.

Comparison Logic

Your metrics only mean something when read as deltas, not absolute values. A standalone RPS of 0.203 tells you nothing. RPS of 0.203 vs. baseline 0.212 tells you your model improved by 4.2%.

Your five model variants are tested on the same locked 64-match test set (WC 2022). Never re-tune on the test set. Parameters h and δ are tuned on train_matches.csv only.

Lesson 2 · ~2 hours
Build the Ablation Loop
Extend your baseline into a full 5-variant comparison table

Goal

Produce one DataFrame with 5 rows (model variants) and columns for RPS, Log Loss, and 95% CI. This is the core deliverable of your project — the academic claim is that each modification improves the baseline, and this table is the proof.

🎯 Concept — What is an ablation study?
An ablation study adds one ingredient at a time and measures the effect of each. The key rule: every variant must be tested on the same held-out test set. Your 64 WC 2022 matches are locked — touch them only to evaluate, never to choose parameters.

Task 1 — Implement each variant as a function

  • Write one probability function per variant:
  • get_home_away_probs(elo_home, elo_away, h) — adds home field offset h to dr
  • get_goal_margin_probs(elo_home, elo_away, goal_diff) — scales importance by G
  • get_draw_zone_probs(elo_home, elo_away, delta) — widens/narrows the draw zone
  • get_full_model_probs(elo_home, elo_away, goal_diff, h, delta) — all three combined

Task 2 — Tune h and δ on the training set

  • Minimize RPS on train_matches.csv only — never touch the test set during tuning
from scipy.optimize import minimize

train = pd.read_csv('train_matches.csv').dropna(subset=['elo_home','elo_away'])

def train_rps(params):
    h, delta = params
    probs = [get_full_model_probs(r.elo_home, r.elo_away,
             r.home_goal_diff, h, delta) for _, r in train.iterrows()]
    result = rps_series(train['outcome'].tolist(),
             [p[0] for p in probs], [p[1] for p in probs], [p[2] for p in probs])
    return result['rps_mean']

opt = minimize(train_rps, x0=[50, 0.3], method='Nelder-Mead')
best_h, best_delta = opt.x
print(f'Best h={best_h:.1f}, delta={best_delta:.3f}')

A reasonable search range: h ∈ [0, 100], δ ∈ [0.1, 0.5]. Tune them jointly for the full model, independently for their respective single-variant rows.

Lesson 3 · ~2–3 hours
Add Calibration Metrics
Prove your probabilities are trustworthy, not just accurate

Goal

Compute ECE for each variant, plot reliability diagrams for the Draw class specifically, and show that the δ parameter reduces Draw ECE. This is the direct visual proof that your draw zone modification is doing meaningful work.

🎯 Concept — Calibration vs. Accuracy
A model can have good RPS but still be wrong in a systematic way. If your model says "70% home win" but home teams only win 55% of those games, it is overconfident. ECE measures this gap across all probability bins. A perfectly calibrated model has ECE = 0.

Task 1 — Implement ece_multiclass()

  • Add to eval_metrics.py — compute ECE per class: Home Win, Draw, Away Win
  • Use 10 equal-width bins · return ece_home, ece_draw, ece_away, ece_mean
def ece_multiclass(outcomes, p_home_list, p_draw_list, p_away_list, n_bins=10):
    from sklearn.calibration import calibration_curve
    import numpy as np
    probs = [p_home_list, p_draw_list, p_away_list]
    names = ['home', 'draw', 'away']
    result = {}
    for cls, (name, p_list) in enumerate(zip(names, probs)):
        y_bin = [1 if o == cls else 0 for o in outcomes]
        frac, mean_p = calibration_curve(y_bin, p_list, n_bins=n_bins)
        result[f'ece_{name}'] = float(np.mean(np.abs(frac - mean_p)))
    result['ece_mean'] = np.mean([result['ece_home'],
                                  result['ece_draw'], result['ece_away']])
    return result

Lesson 4 · ~2 hours
Brier Score Decomposition
Separate calibration errors from discrimination errors

Goal

Decompose the Brier score into three interpretable components for each model variant. This answers: is the model getting better because it's more honest, or because it's more discriminating?

ComponentWhat it measuresDirectionVaries across variants?
REL (Reliability)Calibration error — how far probs are from observed frequenciesLower ↓Yes
RES (Resolution)Discrimination — how much predictions deviate from base rateHigher ↑Yes
UNC (Uncertainty)Difficulty — set by the outcome distribution in the test setFixedNo

Key formula: Brier Score = REL − RES + UNC

Task 1 — Implement brier_decomposition()

  • Add to eval_metrics.py
def brier_decomposition(outcomes, p_home, p_draw, p_away):
    import numpy as np
    o_mat = np.eye(3)[outcomes]               # (N x 3) one-hot
    p_mat = np.column_stack([p_home, p_draw, p_away])
    base_rate = o_mat.mean(axis=0)            # climatological frequencies
    bs  = np.mean(np.sum((p_mat - o_mat)**2, axis=1))     # total Brier Score
    unc = np.sum(base_rate * (1 - base_rate))              # uncertainty (fixed)
    rel = np.mean(np.sum((p_mat - base_rate)**2, axis=1))  # reliability
    res = unc + rel - bs                                    # resolution
    return {'BS': round(bs,5), 'REL': round(rel,5),
            'RES': round(res,5), 'UNC': round(unc,5)}

Task 2 — Validate UNC is constant

  • UNC must be identical across all 5 rows — if it differs, there is a bug
  • REL should decrease Baseline → Full Enhanced
  • RES should increase Baseline → Full Enhanced
⚠️ Watch for this failure mode
If REL drops but RES also drops, the model is better calibrated but less discriminating. This is a legitimate finding — report it honestly rather than burying it.

Lesson 5 · ~2 hours
Fairness Slices
Answer whether the model is fairer across confederations

Goal

Compute RPS broken down by confederation for each model variant — the quantitative fairness argument. Your enhanced model either supports or complicates the claim that FIFA disadvantages non-UEFA teams. Report honestly either way.

🎯 Concept — Why sliced metrics?
A single global RPS averages over 64 matches. But your test set has ~30 UEFA matches and only ~8 CAF matches. If your model performs great on UEFA games but poorly on CAF games, the global number masks this. Sliced metrics reveal where the model works and where it doesn't.

Task 1 — Slice RPS by confederation

  • Group test matches by home_conf / away_conf and compute RPS per group
  • Report n alongside each RPS — slices with n < 5 are too small to report
from eval_metrics import rps_series

confederations = test['home_conf'].dropna().unique()
results = {}
for conf in confederations:

from scipy.stats import spearmanr

elo_snap = pd.read_csv('elo_2025_snapshot.csv')
elo_snap['model_rank'] = elo_snap['total_points'].rank(ascending=False).astype(int)

fifa_latest = fifa[fifa['rank_date'] == fifa['rank_date'].max()]

merged = elo_snap.merge(fifa_latest[['country_full','rank']],

🎯 Concept — The Skill Score
Instead of reporting raw RPS, report RPS_skill = 1 − (RPS_model / RPS_baseline). This converts your result into a percentage improvement that anyone can interpret immediately. A skill score of 0.085 means "8.5% improvement over baseline" — one sentence that answers Heilmeier Q5.
